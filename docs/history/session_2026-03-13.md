## What We Did Today

### 1. Time/Duration Formatting Bug — FIXED (partially)
The AI was storing decimal float values (e.g., `6.133333...` for 6h 8m) and the frontend was displaying them raw. Created a shared formatting utility at:

**`/src/lib/utils/format.ts`** — `formatFieldValue(value, unit, label)`

- Handles time labels (start/end → `HH:MM`)
- Handles duration labels (awake/rem/light/deep/actual sleep → `Xh Ym`)
- Handles string → number coercion (values stored as strings in DB)
- Plug this into **every** component that displays tracker field values

Integrated into:
- `ActionCard.tsx` (pending log card in chat)
- `TrackerDayGroup.tsx` (journal view)
- `LogEntryCard.tsx` (tracker history view)
- `TrackerHistoryView.tsx` (daily stats footer)

**Status:** Formatting looks correct in the pending card (sc1) and journal. The individual tracker detail page (sc2) still shows raw decimals from the DB because that component uses `LogEntryCard` — just integrated the formatter there too, should be fixed.

### 2. Sleep Score / Time In Bed Swap — FIXED
The AI swaps Sleep Score and Time In Bed because the DB schema fieldIds are semantically reversed (legacy naming). Two-layer fix:

- **Smart Swapper in `sanitizeFields`**: detects if a score field has a duration-like value and vice versa, swaps them.
- **Schema Enforcement in chat route**: after Gemini returns actions, the route now **overwrites** the AI's `fieldLabels` and `fieldUnits` with the authoritative DB schema values. Gemini's guesses are discarded.

**Status:** The pending card (sc1) now shows correct values. The DB may still have old swapped data from previous bad saves — those historical entries will show wrong values until manually corrected or deleted.

### 3. Attachment Persistence — DONE
Added `attachments` JSONB column to `chat_messages` table. Attachments now persist after send and display in chat history.

### 4. Chat Route Performance
Moved `getTrackersBasic()` and `getAgents()` to parallel `Promise.all` fetch. Removed the heavy log-summary logic from the AI prompt-building path.

**Status:** Some improvement but the Gemini API call itself is the main bottleneck (~1-2s). The route still does 4-5 sequential async steps before Gemini is called (session fetch, routine fetch, agent fetch, save user message, build prompt). Further optimization possible.

### 5. Journal Layout & Correlator Refinements — DONE
- **Rebuilt Journal**: Two-pane layout with date sidebar and header bar (View/Correlator buttons).
- **Correlator Modal**: Modal with existing metrics list and a formula builder.
- **UI Tweaks**: Changed the multiplication symbol from `*` to `×` as requested.

### 6. Dashboard Routine Logic — DONE
Implemented the "Start Day" and "End Day" banner visibility rules:
- **Start Day**: Shows if not started today OR if day was ended today (to allow starting "tomorrow" if staying up late).
- **End Day**: Shows from 7 PM (19:00) onwards if not already completed for the day.
- **State Tracking**: Created `user_day_state` table to persist these completions.
- **Automatic Marking**: The chat route now automatically marks the day as started/ended when you finish the final step of the respective routine.
